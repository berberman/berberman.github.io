shout 接收三个参数：类型变量 a、类型类实例 Show a 以及 x。类型类实例有点像隐式转换，编译器会在上下文中去找这个东西，使得函数体中能够正确调用类型类约定的函数。
值得注意的是和类型变量不同，类型类实例只能靠编译器推断，不能手动传进去。

三类形参

下面介绍三种形参具有的特性：dependent、relevant、visible。

  • dependent —— 这个参数是依赖的，即这个参数值的选择会影响后续的参数。
  • relevant —— 这个参数是相关的，即这个参数值能在运行期函数体中使用。
  • visible —— 这个参数是可见的，即这个参数能显式传入。

这样我们可以画出一张表（在常规情况下）：

类型变量 类型类实例 通常的项
dependent 是 否 否
relevant 否 是 是
visible 否 否 是

通过上面 const 与 shout 的例子，我们得知了（在常规情况下）：

  • 通常项，例如 Int -> ... 是 relevant（参数就要在函数体用嘛）、visible（当然要显式传，不然怎么知道是几）并且 non-dependent（你 Haskell 并没有原生依赖类型）的。

  • 类型类实例，例如 Show a => ... 是 relevant（函数体可以调用到类型类实例中的代码）、invisible （前面说过了，只能靠编译器找）并且 non-dependent 的。

  • 类型变量，例如 ∀ (a :: Type). ... 是 irrelevant、invisible 并且 non-dependent 的。

有了常规情况那肯定就有非常规情况，咱前面就已经试过把类型变量当显式参数传了（TypeApplications，虽然并不能算 visible），那么我们暂且画出一张更全的表：
